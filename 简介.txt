# WebScribe · 智能网页探索与复刻工具  
## 功能描述文档（自然语言版）

---

## 一、这是什么？

**WebScribe** 是一款能够“自己看懂”任何网页的智能工具。你只需要给它一个网址，它就会像一位资深开发人员一样，自动打开网页、四处点击、填写表单、滚动页面，全面探索网站的所有功能。在探索过程中，它还会默默记录下所有网络请求、页面变化、设计细节，并最终为你生成一套可以直接使用的开发材料——包括可运行的前端代码、API文档、设计系统规范、性能报告，甚至是一张清晰的页面跳转关系图。

简单来说，WebScribe 帮你把“分析一个陌生网站”这件事，从几小时甚至几天的逆向工程，压缩成一次点击。

---

## 二、它能做什么？

### 1. 自主探索——像人一样浏览网站
- 打开你指定的网址，自动识别所有按钮、链接、可点击区域。
- 像真人一样点击、输入、滚动、悬停，触发隐藏菜单、弹出窗口、懒加载内容。
- 深度遍历整个网站，不放过任何一个页面和状态。
- 如果遇到登录页面，你可以预先提供账号密码，WebScribe 会自动填写并登录（还能识别验证码，或者请你帮忙输入一次）。
- 你可以随时通过网页控制台暂停、跳过某个元素、甚至停止探索，完全掌握主动权。

### 2. 全量捕获——不放过任何细节
探索过程中，WebScribe 会默默记录下一切：
- **网络请求**：页面加载时发出的所有 API 调用（XHR、Fetch），包括请求地址、参数、返回数据、耗时。
- **WebSocket 消息**：实时通信的内容（如聊天消息、股票行情）。
- **页面变化**：DOM 结构如何动态增减，哪些元素被添加或移除。
- **设计系统**：页面上用了哪些颜色、字体、字号、圆角、间距，并整理成一份可直接使用的 CSS 变量。
- **性能指标**：页面加载花了多久？DNS 解析慢不慢？哪些资源太大？自动生成优化建议。
- **无障碍问题**：检测是否缺少图片 alt、颜色对比度不足等，输出 WCAG 合规清单。
- **媒体资源**：页面中的图片（包括背景图）、视频、音频链接，自动提取并分析格式、尺寸。
- **文字内容**：主要段落、文章正文，用于内容摘要或 SEO 分析。

### 3. 智能生成——直接给你可用的开发材料
探索完成后，WebScribe 会为你打包一份完整的产出物，包括：

#### 前端代码（可选）
- 根据捕获的页面结构和网络请求，自动生成一个 **React / Vue / Svelte** 项目，包含：
  - 完整的项目文件结构（`package.json`、`src/App.jsx`、`src/index.js` 等）
  - 页面布局组件，使用提取的设计 token 定义样式
  - API 服务层，用 Mock 数据模拟后端接口
  - 交互逻辑（点击、表单提交）的完整实现
- 生成的代码可以直接运行，帮你快速搭建一个和原网站功能相似的 Demo 或新应用。

#### API 文档
- 将所有捕获的请求整理成 **OpenAPI 3.0** 规范文件（JSON 或 YAML），可直接导入 Postman、Apifox 等工具，或用于自动生成前端/后端 SDK。

#### 设计系统
- 输出一份 CSS 变量文件（`design_tokens.css`），包含页面中使用的颜色、字体、圆角等，方便你在新项目中保持一致的设计风格。

#### 移动端适配
- 分析页面中已有的媒体查询，自动生成一份移动端适配 CSS，帮助你的项目快速适配手机屏幕。

#### 状态图谱
- 用 Mermaid 流程图画出页面的跳转关系，直观展示你探索了哪些页面、如何跳转。

#### 综合报告
- 一个漂亮的 HTML 报告，汇总所有探索数据：访问页面数、请求列表、性能评分、无障碍问题、设计系统、截图等，方便分享或归档。

## 三、怎么使用？

### 界面简洁，操作简单
WebScribe 提供了一个 Web 控制台（网页界面），你只需要：

1. **创建任务**：输入目标网址，选择探索深度（比如最多点几层）、最大页面数，选择反爬策略（开发/标准/隐匿/激进），还可选是否启用 AI 生成代码、是否提取内容、是否提供登录账号等。
2. **实时监控**：点击开始后，控制台会实时显示探索日志，你能看到它正在访问哪个页面、点击了哪些元素、捕获了哪些请求。你还可以随时点击“暂停”、“继续”、“跳过当前元素”、“停止”，灵活控制探索过程。
3. **下载成果**：任务完成后，点击“下载报告”按钮，即可获得一个 ZIP 压缩包，里面包含所有生成的文件。

### 支持多种部署方式
- **一键启动**：使用 Docker Compose，只需运行一个脚本，所有服务（后端、前端、Redis）就自动启动。
- **手动部署**：也可以分别启动后端和前端，适合开发和调试。
- **云端部署**：可以部署到任意支持 Docker 的云服务器，或使用 Kubernetes 进行大规模分布式探索。

## 四、为什么它很强大？

### 1. 反反爬，不易被屏蔽
- 内置四种反爬策略，从“开发模式”（完全无伪装）到“激进模式”（代理轮换、指纹随机化、模拟人类点击），轻松应对大多数网站的检测。
- 随机 User-Agent、随机鼠标轨迹、随机延迟，让自动化行为更像真人。

### 2. 稳定可靠
- 对关键操作（点击、导航）实现自动重试，网络波动时也不会轻易失败。
- 浏览器崩溃后会自动重启并恢复任务，不会丢失已探索的进度。

### 3. 灵活扩展
- 支持分布式部署：使用 Redis 作为任务队列，可以同时运行多个探索任务，甚至多台机器协同工作，适合大型项目或批量分析。
- 模块化设计：所有功能（反爬、AI、内容提取等）都可以独立配置或替换，方便你根据自己的需求定制。

### 4. 隐私与安全
- 所有数据默认存储在本地，不上传云端（除非你主动使用外部 AI 服务）。
- 登录凭据仅在任务运行时使用，完成后可一键清除。
- 支持加密存储敏感信息（如 API Key）。

## 五、适合谁用？

- **前端开发者**：想学习某个网站的交互实现，但又不想一行行看源码？让 WebScribe 帮你生成一个可运行的简化版，边看边学。
- **后端开发者**：需要为一个已存在的网站开发新功能，但不知道它的 API 怎么用？WebScribe 会帮你整理出完整的 API 文档。
- **产品经理 / 设计师**：想了解竞争对手的设计风格和交互流程？WebScribe 能提取设计 token，并生成状态图，让你一目了然。
- **测试人员**：需要编写 UI 自动化测试用例？WebScribe 生成的选择器和操作序列可以直接用于 Playwright 或 Puppeteer 脚本。
- **创业者**：想快速复刻一个成熟网站的核心功能？WebScribe 生成的代码可以帮你节省大量时间。

## 六、举个例子

假设你想分析一个电商网站的“商品详情页”和“加入购物车”流程。

1. 打开 WebScribe 控制台，输入网址 `https://shop.example.com/product/123`。
2. 配置最大深度为 2，最大页面数为 30，选择“激进”反爬策略，并开启“AI 代码生成”。
3. 点击开始探索。WebScribe 会自动：
   - 进入商品详情页，点击“加入购物车”按钮；
   - 捕获“加入购物车”的 API 请求（POST /cart/add）及其响应；
   - 进入购物车页面，点击“结算”；
   - 遇到登录页时，根据你提前填写的账号密码自动登录；
   - 记录所有页面的颜色、字体、圆角；
   - 最后生成一个 React 项目，包含商品详情页、购物车页、结算页的组件，并模拟了 API 响应。

4. 下载报告，你得到一个 ZIP 包，内含：
   - 前端代码（可直接 `npm start` 运行）
   - OpenAPI 文档（可导入 Postman）
   - 设计系统 CSS 变量
   - 状态图（展示“商品详情→购物车→结算”的跳转）
   - 性能分析（如加载时间、资源大小）
   - 无障碍报告（如缺少图片 alt 等问题）

整个过程不到 10 分钟，而如果手工逆向，可能需要数小时甚至数天。

## 七、部署选项

- **本地单机**：使用 Docker Compose，一行命令启动所有服务，适合个人或小团队。
- **私有云**：将镜像推送到私有仓库，在云服务器上运行，数据完全可控。
- **分布式集群**：使用 Kubernetes，配合 Redis 任务队列，支持大规模并发探索，适合企业级应用。
- **桌面应用**：未来计划提供 Electron 打包版本，无需安装 Docker，双击即可运行。

## 八、总结

WebScribe 不是一个简单的网页抓取工具，而是一个**能够理解网页的智能助理**。它把复杂的逆向工程自动化、标准化，让开发者能够聚焦于创新，而不是重复劳动。无论你是想学习新技术、做竞品分析、还是加速二次开发，WebScribe 都能成为你的得力助手。
